Insurance Claim Prediction Classification Trees

Lance Dugger

Problem Description

The insurance training data set is a csv file consisting of 25 columns that describe different customer attributes for an insurance company. The target column includes whether these customers filed a claim or not. Classification tree methods will be used to develop a model that will predict whether a customer will file a claim or not.

Data Preparation

# Fix NA's, code necessary variables
data$AGE[is.na(data$AGE)] <- mean(data$AGE, na.rm = "TRUE")
data$YOJ <- na.aggregate(data$YOJ, data$JOB, mean, na.rm = TRUE)
data$INCOME <- na.aggregate(data$INCOME, data$JOB, mean, na.rm = TRUE)
data$HOME_VAL <- na.aggregate(data$HOME_VAL, data$JOB, mean, na.rm = TRUE )
data$CAR_AGE <- na.aggregate(data$CAR_AGE, data$CAR_TYPE, mean, na.rm = TRUE)
data$CAR_AGE[data$CAR_AGE < 0 ] <- 0 
data$OLDCLAIM <- ifelse(data$CAR_AGE < 5 & !is.na(data$CAR_AGE),0,data$OLDCLAIM)
data$OLDCLAIM <- na.aggregate(data$OLDCLAIM, data$CAR_AGE, mean, na.rm = TRUE )
data$HOME_OWNER <- ifelse(data$HOME_VAL == 0, 0, 1)
data$YOUNG <- ifelse(data$AGE < 29, 1, 0)
data$HAS_KIDS <- ifelse(data$HOMEKIDS == 0, 0, 1)
data$OLD_CAR <- ifelse(data$CAR_AGE > 20, 1, 0)
data$COLLEGE <- ifelse(data$EDUCATION == "Bachelors" |data$EDUCATION == "Masters" | data$EDUCATION == "PhD", 1, 0)
data$NEW_JOB <- ifelse(data$YOJ < 8, 1, 0)
data$MANAGER <- ifelse(data$JOB == "Manager", 1, 0)
data$CHEAP_CAR <- ifelse(data$BLUEBOOK < 6000, 1, 0)

# Bin Income
data$INCOME_bin[is.na(data$INCOME)] <- "NA"
data$INCOME_bin[data$INCOME == 0] <- "Zero"
data$INCOME_bin[data$INCOME >= 1 & data$INCOME < 30000] <- "Low"
data$INCOME_bin[data$INCOME >= 30000 & data$INCOME < 80000] <- "Medium"
data$INCOME_bin[data$INCOME >= 80000] <- "High"
data$INCOME_bin <- factor(data$INCOME_bin)
data$INCOME_bin <- factor(data$INCOME_bin, levels=c("NA","Zero","Low","Medium","High"))

Model Selection

Four different methods were used to model the insurance data. The first method was a classification decision tree. This method predicts an outcome based on a series of splits that are determined by the different predictor variables. This method had 73% total accuracy, however it significantly underperformed in identifying clients who would file a claim. The second method was bagging. This uses decision trees like the first method, however it uses random sample sets to generate many different trees, which decreases the variance of the final output. The third method was a random forest. This method is very similar to bagging, except that a random subset of predictor variables are used to split each of the trees. This helps to decorrelate the trees. The final method used was a gradient boosting machine. This method builds multiple trees, except that each tree built attempts to correct the errors of the previous tree. Bagging, random forest, and gradient boosting all produced models with similar performance. The areas under the curve for their receiver operating characteristics curves were within 0.03 of eachother and their accuracies were within 0.03 of eachother. Both of the previously stated metrics are measured on a scale between zero and one. Model Two, the bagging method, was chosen because it had the highest rate of detecting claims, which is the ultimate goal of the insurance agency.
